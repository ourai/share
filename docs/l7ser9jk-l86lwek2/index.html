术语

一些不容易理解的词汇解释：

数据项与字段

在本系统中，「字段」这个词广义上是指实体的属性，狭义上则是值为简单数据类型的实体属性，即非关系字段，如：字符串、数值、布尔等。在产品层面以狭义上的含义作为解释。

「数据项」这个词在作为根节点时代表实体，而在作为其他节点时表示关联了实体的关系字段（此「字段」为广义上的含义）：一对一、一对多、多对一、多对多。在产品层面合并简化为对象和列表。

设计

用户与权限

权限控制

Handie 底层对操作权限的控制，即通过视图描述器配置的动作是否展示，是根据视图部件中获取到的权限 map 来的，要在登录后获取用户信息的请求函数中将可访问的权限处理好返回给视图层。

由于后端没有定义细粒度的权限，只是根据用户身份粗略地来，因此要在前端设计出相对细粒度的权限：

前端权限 用户身份（角色） 实际用户权限
spaceAdmin:read superAdmin 可访问空间及空间管理员管理相关页面
spaceAdmin:edit superAdmin 可进行空间及空间管理员管理相关操作
spaceMember:read admin 可访问所管辖空间的成员管理相关页面
spaceMember:edit admin 可进行所管辖空间的成员管理相关操作
space:read everyone、write、admin 可访问所属空间的常规功能相关页面
space:edit write、admin 可进行所属空间的常规功能相关操作

除了超级管理员，其他用户都得被加入空间后才能访问除登录页和首页之外的页面。

领域建模

类型定义：

新增/编辑表单

采用在对话框中左树右表单的两栏布局的交互形式：

其中，红框部分是最外层的表单对话框视图，绿框部分是展示为树的 children 字段，蓝框部分则是表单对话框视图中内嵌的用于新增/编辑数据项/字段信息的表单视图。

由于数据项配置信息的前后端数据结构不一致，前端是有父节点标记的平铺数组，而后端是树形结构，故而在前后端交互时需要分别进行正反向转换。

在界面中交互时的数据流为——

最外层的表单对话框将数据项配置信息下发到配置树字段；配置树字段在初始化及字段值发生变动时构造展示用的树形数据结构；树节点在新增/编辑时将对应信息经由外层表单传给内嵌表单，点击「保存」按钮后再由外层表单更新数据项配置信息。

可简单总结为：配置树字段负责展示；内嵌表单用于编辑；外层表单主要起到桥接作用。
