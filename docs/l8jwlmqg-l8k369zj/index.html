基于文件系统的数据管理

欧雷 发表于 2022-09-27 15:55:19

背景

博客、API 文档等的最终产物都是静态页面，它们的数据源就是符合一定标准、格式的本地文件——可将这些文件视为元数据与数据记录的混合体。

当文件多了、站点复杂了之后，单纯的本地文件形式存在几个重要痛点：

  • 无法方便地知道自己有哪些文档；
  • 查找与编辑不方便；
  • 文档间的关系不明了也无法管理。

需要一种方式，能够像中后台业务系统一样管理本地文件，并能够通过某种方式将本地文件的数据进行转换。

定义

数据定位

通过文件夹的层级控制去表达数据的「集合」、「记录」等概念。

集合 是一个以英文单词的复数形式命名的文件夹，如 posts；记录 是一个遵从一定规则的可多层级的文件夹，如 2022/09/27-155519。

元数据

在 集合 或 记录 文件夹下的 .meta 文件夹中存放元数据，可以是 YAML、Markdown、JS 等文件。

数据记录

记录 文件夹下的 readme.md 文件作为 content 字段，其他字段的读取/存储规则为：

  • 读取 metadata.yml 或 basic.yml 作为基础字段；
  • 其他 Markdown 文件及子文件夹下的文件作为扩展补充字段，若基础字段中有同名的则覆盖。
